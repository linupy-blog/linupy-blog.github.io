完全用 arch 工作

本文定位: 帮助你转到 arch_linux 作为个人桌面系统(由于滚动安装所以不适合生产环境),如果你已经是 linux 或者 mac 用户的话.(如果你是 win 用户,试试 linux 也好,如果你觉得 linux 丑陋那么用 OS X 也是一个很好的选择).

你需要知道: 使用过 linux,可以大致看懂命令的意义.

写在前面的话

  • 如果你是 linux 老鸟的话，看到这个题目就已经可以想到那个人了。虽然后来他也承认那是一时冲动。但我认为现在是完全有可能的.本文的题目绝不是在虚张声势,我本人就完全使用 arch : )
  • 一直以来都把 linux 当作玩物，说实话 linux 与 OS X 相比的确有很大的差距（就易用性和方便程度来说，可能有狂热的 linux 爱好者不同意我说的，但这就是事实，当你提出来），当然垃圾 windows 不再比较范围之内.
  • 但是，在某一个无眠的夜晚，突然又翻到 GNU 宣言，突然被感动，决心放弃 OS X，然后迅速把 mac 出手，买了这台 P50.声明:我不是狂热的自由软件分子
  • linux不是最好的,但比win要优秀的多.就像民主之于专制.
  • 因为 arch 的 aur 有非 GNU 的软件,所以不被 GNU 认可
  • 总是会有人说: linux 不如什么什么好用,当然这有一部分是事实,linux 本身就不是为了日常使用设计的.但是我们需要一种选择,一种知情权,我们需要知道我手上的这个黑盒子在干什么.

arch 的优势所在

  • arch 的包实在是太丰富了,基本你能想到的都能直接在 AUR 中找到,大概是 ubuntu 的 1.5 倍之多.
  • arch 占用资源很少,我开着终端,音乐,firefox,总共不要 2G 内存.适合配置不高的电脑安装.
  • 用 arch 是自由的,有一些基于 arch 的开发版把一切都装好了,但这恰恰和 arch 的开发理念背道而驰,arch 的 wiki 中有非常详尽的关于各个阶段工具的选择.
  • 当然还有一些 linux 共有的优势,这里就不一一列举了.

关于安装

其实传言中说的 arch 安装很难是个谣言，都是那些自以为是的家伙的莫名其妙的优越感在作祟（看我们在用 arch，我们可是专家：） 我前前后后安装了有十几次，其实也就是比一般的 linux 安装（比如 cent的命令行安装）多了包选择，引导等。按照官方 wiki，即使你不明白发生了什么，也可以很容易的完成系统安装或者可以看萌狼的教程，如果你会使用搜索引擎的话。所以你不会从本文学到如何安装 arch；）

基本包的选择

这些包基本上都是我在用的,也是用过中体验最好的,如果你有更好地选择,可以邮我.

建议安装 base linux-lts linux-firmware networkmanager

  • linux-firmware 包含了必要的固件（bluetooth wifi 等），如果一开始不安装，等安装完成没有网络很难解决问题。
  • 关于内核版本建议选择长期支持版。

引导程序： systemd-boot

现在 arch 可以用的引导程序可以在这里看到.之所以选择 systemd-boot 是因为不需要额外安装，这符合 arch 的哲学.而且他的启动界面比 grub 要好看一些。

systemd-boot 只支持引导 efi 分区启动，所以你要保证主板的 efi 启动开启，且挂在 efi 分区。

  1. bootctl --path=/boot install path 后面应该是你的 efi 分区，也可能是/boot/efi.

  2. 接下来需要编辑两个文件：

/loader/loader.conf：

bash default arch # 引导界面的默认项 timeout 4 # 引导界面等待时间 console-mode max editor no # 为防止别人修改，建议填no

/boot/loader/entries/arch.conf :(默认没有需要创建)

title   Arch Linux            # 启动时显示名称
linux   /vmlinuz-linux        # 以下路径都是以/boot为根
initrd  /intel-ucode.img      # 这是微码，需要手动安装：比如intel: pacman -S intel-ucode

initrd  /initramfs-linux.img
options root=/dev/sda2 rw     # efi分区位置
  1. 保存退出.

登录管理器: sddm

  1. 安装: yay -S sddm sddm-theme-sugar-candy-git

  2. 生成配置文件: # sddm --example-config > /etc/sddm.conf

  3. 修改 /etc/sddm.conf

bash [Theme] Current = sugar-candy

程序

下面的程序都是我用过的最好的,有一些讲了基本的配置.如果你有补充或者更好的选择欢迎联系我.

软件包管理： yay

现在 arch 有好几个包管理工具，之所以选择 yay 是因为它字母少；p

安装：

git clone https://aur.archlinux.org/yay.git
cd yay
makepkg -si

包转换: debtap

如果程序只有 deb 包的话,可以通过这个转换:

yay -S debtap
debtap -u  #update
debtap packagename

WM or DM : i3-gaps + i3-blocks 或者 xfce4

就我的使用体验来说,配置好的 i3 是要比 xfce4 好用许多.但是如果你不喜欢使用平铺式桌面也可以用 xfce4.

i3 配置:

配置参考了这个和yanji的配置,加上一些自己的习惯.

  1. 安装必要的包
 yay -S i3-gaps i3blocks rofi picom feh xfce4-battery-plugin xfce4-power-manager nerd-fonts ranger arc-gtk-theme papirus-icon-theme lxappearance-gtk3 
 ```

 最后这个包可以图形化修改主题和图标.

2. `git clone https://github.com/da-edra/dotfiles.git`

3. 复制对应的配置文件到`~/.config`

安装字体拓展: `yay -S nerd-fonts`

设置透明度: compton 前端: compton-conf

## 声音

1. 安装 `alsa-utils` 解除声音屏蔽
2. 通过`alsamixer` 调节音量输出

### 蓝牙

1. 安装 `gnome-shell blueman #blueman 提供后端 前端 托盘`
2. 在`.config/i3/config`中添加: `exec --no-startup-id blueman-daapters`以开机启动
3. 如果想要通过蓝牙耳机输出还需要
   1. `yay -S pulseaudio-bluetooth pulsemixer`
   2. 通过`pluseeaudio` 设置蓝牙为默认.

### 终端模拟器 : termite

### shell: zsh + fish

fish 在不配置的情况下也非常好用,但是由于语法和 bash 不同,所以搭配 zsh 一起使用.[这里](https://www.ruanyifeng.com/blog/2017/05/fish_shell.html?from=groupmessage&isappinstalled=0)有阮一峰的教程.

#### zsh 配置:

```bash
yay -S zsh zsh-theme-powerlevel9k
sudo chsh -s /bin/zsh username      #设为默认shell
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"        #安装oh-my-zsh
echo 'source /usr/share/zsh-theme-powerlevel9k/powerlevel9k.zsh-theme' >> ~/.zshrc
# 安装pwoerlevel9k主题

输入法： rime

不管你使用简体繁体，五笔还是双拼，rime 都是你最好的选择，当然如果你想让他和你心意，需要花时间在文档和配置上，不过如果你不想在这上面花费时间，rime_pro也是个很好的选择。

  1. 安装 rime： yay -S ibus-rime

  2. 开机自启动：

在~/.xprofile 中添加：

bash export GTK_IM_MODULE=ibus export XMODIFIERS=@im=ibus export QT_IM_MODULE=ibus ibus-daemon -x -d

  1. 配置：

下载前面提到的 rime_pro 增强包，解压到对应的目录，重新部署即可. 当然你也可以自己配置。
